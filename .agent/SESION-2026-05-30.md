# Sesión - 30 Mayo 2026

## Stack
Astro 5 (static) + React + Tailwind v4 + Sveltia CMS

## Problema
Sitio estático (sin SSR ni DB), la presidenta de la asociación quiere poder actualizar eventos, noticias, equipo, etc. por su cuenta.

## Solución implementada
Sveltia CMS (v0.37.5) en `public/admin/` → CMS visual, auth por GitHub, commitea directo al repo.

## Pendiente para terminar

### 🔴 En GitHub (Settings → Secrets and variables → Actions):
- `VPS_SSH_KEY`: private key SSH del VPS
- `VPS_HOST`: IP o dominio del VPS
- `VPS_USER`: usuario SSH (ej: root)

### 🔴 Cuenta de GitHub para la presidenta:
- Crearle cuenta, agregarla como collaborator al repo
- Ella entra a `https://asociacionsimisumaq.org/admin`, login con GitHub, ya puede editar

### 🟡 Posibles mejoras futuras:
- Migrar más páginas hardcodeadas (que-hacemos, servicios, etc.) a CMS
- Widget `markdown` para descripciones si prefiere markdown sobre HTML

### Flujo final
Presidenta edita en `/admin` → Sveltia CMS commitea a `main` → GitHub Actions build ea y scp al VPS → sitio actualizado. Sin tocar código.
